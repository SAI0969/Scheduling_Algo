CPU Scheduling Algorithms

Introduction

What is CPU Scheduling?

Whenever the CPU becomes idle, the operating system must select one of the processes in the ready queue to be executed. The selection process is carried out by the short-term scheduler, or CPU scheduler.


Why Scheduling is needed

It is desirable to maximize CPU utilization and throughput and to minimize turnaround time, waiting time, and response time.


Objectives of Scheduling

High CPU utilization, low waiting time, low turnaround time, good response time and fairness.

Key Terms

Arrival Time (AT)

It is the time at which a process enters the ready queue and becomes available for execution by the CPU.

Burst Time (BT)

It is the total amount of CPU time required by a process to complete its execution.

Completion Time (CT)

It is the time at which a process finishes its execution completely.

TurnAround Time (TAT)

It is the total time taken by a process from its arrival in the ready queue until its completion of execution.

Formula:
TAT = CT - AT

Waiting Time (WT)

It is the total time a process spends waiting in the ready queue before getting CPU execution.

Formula:
WT = TAT - BT
or
WT = CT - AT - BT

Response Time (RT)

It is the time between the arrival of a process and the first time it gets CPU execution.

Formula:
RT = First CPU Allocation Time - AT

Note: Most of the time AT, BT, CT, will be mentioned in the question.

Scheduling Criteria

Throughput

Number of processes completed per unit time.

Turnaround Time

Total time from submission to completion.

Waiting Time

Time spent waiting in ready queue.

Response Time

Time until first response is produced.

CPU Utilisation

Keep CPU busy as much as possible.

Scheduling Algorithms

First Come First Serve (FCFS)

Click to view FCFS

It is non-preemptive.

By far the simplest CPU scheduling algorithm. The process that requests the CPU first will be allocated the CPU first.

The implementation of the FCFS policy is easily managed by a FIFO queue.

The job which comes first in the ready queue will get the CPU first. FCFS scheduling algorithm simply schedules the jobs according to their arrival time.


ADVANTAGES: Easy to implement.
DISADVANTAGES: Average waiting time under the FCFS policy is often quite long.
Click to view FCFS Solving Steps & Examples
Steps to Solve FCFS

1. Arrange processes in ascending order of Arrival Time (if given).
2. Draw the Gantt Chart.
3. Calculate Completion Time (CT) for each process.
4. Calculate Turnaround Time (TAT = CT - AT).
5. Calculate Waiting Time (WT = TAT - BT).
6. Calculate Average Turnaround Time and Average Waiting Time.

Shortest Job First (SJF)

Click to view SJF

Can be Preemptive or Non-Preemptive.

The process with the smaller burst time is selected for execution next. The main goal of SJF is to reduce average Waiting Time. An appropriate term for this scheduling algorithm is Shortest-Next CPU Burst Algorithm.

Note: If the CPU burst time of two processes is the same, FCFS is used to break the tie.


ADVANTAGES: Minimum Waiting Time, Better Turnaround Time.
DISADVANTAGES: Starvation can occur, Burst Time must be known beforehand.
Click to view SJF Solving Steps & Examples
Steps to Solve SJF

1. Check all processes that have arrived in the ready queue (assume 0ms if AT is missing).
2. Select the process with the shortest burst time.
3. Execute it completely (if Non-Preemptive).
4. Repeat until all processes are complete.

Priority Scheduling

Click to view Priority Scheduling

Can be Preemptive or Non-Preemptive.

A preemptive priority scheduling algorithm will preempt the CPU if the priority of a newly arrived process is higher than the current running process. Non-preemptive will simply put it at the head of the ready queue.


ADVANTAGES: Important processes get processed immediately, supports multi-tier tasks.
DISADVANTAGES: Starvation (Indefinite blocking of low-priority tasks).
Solution to Starvation: Aging. Gradually increasing the priority of processes that wait in the system for a very long time.
Click to view Priority Solving Steps & Examples
Rule: If not stated otherwise, lower priority number = higher actual priority choice.

Steps to Solve Priority Scheduling (Non-Preemption)
1. Create Table with given data.
2. Identify the Priority Rule from question (Smaller number = Higher Priority or Larger number = Least Priority) {If not mentioned, consider smaller number = higher priority}.
3. At Time 0ms, Select process that has highest priority.
4. Since it is Non-Preemption, no interruption is allowed.
5. Draw Gantt Chart.
6. Calculate CT, TAT, WT, AVG TAT, AVG WT.

Steps to Solve Priority Scheduling (Preemption)
1. Create Table with given data.
2. Identify the Priority Rule from question {If not mentioned, consider smaller number = higher priority}.
3. At Time 0ms, Select process that has highest priority.
4. Since it is Preemption, Check every new arrival.
5. Decide whether to preempt that process or not.
6. Draw Gantt Chart.
7. Calculate CT, TAT, WT, AVG TAT, AVG WT.

Round Robin Scheduling Algorithm

Click to view Round Robin

It is purely Preemptive. Designed especially for time-sharing systems.

A small unit of time called a Time Quantum (TQ) or time slice is defined (usually 10 to 100 ms). The ready queue is treated mathematically as a circular queue structure. The CPU scheduler goes around assigning cycles sequentially.


ADVANTAGES: Fair allocation, zero starvation occurrences, predictable response times.
DISADVANTAGES: High context switching overhead calculations. Performance heavily depends on picking the ideal Time Quantum size.
Click to view RR Solving Steps & Examples
Steps to Solve Round Robin

1. Create Table and note Time Quantum (TQ).
2. Create Ready Queue as:
    Time = 0ms => Ready Queue : P1
    Processes arriving later are added to the end of the queue.
3. Execute for One Time Quantum.
4. If Burst Time (BT) = 0 it means that process is completed and remove it from queue, Else Move process to rear of queue.
5. Add Newly Arrived Processes:
    While a process is running, check for new arrivals.
    Add them to the end of the ready queue as they arrive.
6. Repeat, until all the process completes its execution.
7. Draw Gantt Chart.
8. Find CT, TAT, WT, Avg TAT, Avg WT.

MultiLevel Queue Scheduling (MLQ)

Click to view MultiLevel Queue Scheduling
Definition:
Multilevel Queue Scheduling (MLQ) divides the ready queue into multiple separate queues based on process type, priority, or category. Each queue can have its own scheduling algorithm, and processes remain permanently assigned to their respective queue.

Single Ready Queue

P1 → P2 → P3 → P4 → P5

Multilevel Queue Structure

System Queue (Highest Priority)
P1, P2
Interactive Queue
P3, P4
Batch Queue (Lowest Priority)
P5, P6

CPU Selection Order

CPU
▲
System Queue
▲
Interactive Queue
▲
Batch Queue
How MLQ Works

1. Divide processes into multiple queues.
2. Assign a scheduling algorithm to each queue.
3. Select a queue according to queue priority.
4. Execute a process from that queue using its scheduling policy.

Advantages

  • Better organization of processes
  • Fast response for important tasks
  • Supports different scheduling policies
  • Suitable for large systems

Disadvantages

  • Starvation of lower queues
  • Fixed queue assignment
  • Complex implementation
  • Less flexibility

Multilevel Feedback Queue Scheduling (MLFQ)

Click to view Multilevel Feedback Queue Scheduling
Definition:
Multilevel Feedback Queue Scheduling (MLFQ) is an advanced CPU scheduling algorithm in which the ready queue is divided into multiple queues of different priorities, and processes are allowed to move between queues based on their behavior and CPU usage.

Unlike Multilevel Queue Scheduling (MLQ), processes are not permanently assigned to a queue.

Common Structural Mistakes to Avoid

1. Context Switching & Gaps

Forgetting to account for CPU Idle Time when no process has arrived yet. If your completion time ends at 3ms but the next task arrives at 5ms, map an explicit IDLE block from 3ms to 5ms on the Gantt chart timeline.

2. Incorrect Waiting Time (WT)

Blindly calculating WT = CT - BT. This shortcut only yields valid answers if Arrival Time (AT) is zero. Always use the uniform rule:
WT = TAT - BT

3. Queue Mismanagement (Round Robin)

Failing to correctly order newly arriving entries behind preempted returns. When a process finishes its quantum chunk, you must first add any brand new tasks landing at that timestamp to the rear queue before appending the interrupted slice block.
